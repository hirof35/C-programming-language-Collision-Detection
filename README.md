C-Language Collision Detection Algorithms本プロジェクトは、C言語を用いて基本的な幾何学形状（円、四角形、三角形）の衝突判定および包含判定を実装したアルゴリズム集です。ゲームエンジン、物理シミュレーション、あるいは組み込みシステムのセンサー干渉ロジック等に応用可能な基礎数理を扱います。実装されているアルゴリズム1. 円 vs 円 (Distance-Based Detection)2つの円の中心点間の距離を三平方の定理によって算出し、半径の合計値と比較することで衝突を判定します。計算式: $\sqrt{(x_1 - x_2)^2 + (y_1 - y_2)^2} \le r_1 + r_2$特徴: 高精度ですが、sqrt（平方根）による計算コストを考慮し、最適化として「距離の2乗」で比較する手法への拡張も容易です。2. 四角形 vs 四角形 (AABB - Axis-Aligned Bounding Box)回転を考慮しない「軸平行境界ボックス」による判定です。4つの辺の境界条件を論理演算することで、最も計算負荷を抑えた衝突判定を実現します。用途: 物体間の大まかな接触判定（Broad-phase collision）に最適です。3. 点 vs 三角形 (Area-Based Inclusion Test)点が三角形の内部に含まれるかを「面積分割法」を用いて判定します。ロジック: 三角形 $ABC$ 内に点 $P$ がある場合、3つの部分三角形（$PAB, PBC, PCA$）の面積の合計は、元の三角形 $ABC$ の面積と一致するという性質を利用しています。数学的実装: 座標幾何学における三角形の面積公式を用い、浮動小数点精度での内外判定を行います。技術的ポイント構造体 (struct) による抽象化: Circle, Rectangle, Triangle 各形状を構造体で定義し、幾何学的データを管理しやすく設計しています。軽量・高速: 標準ライブラリ（stdio.h, math.h）のみに依存しており、リソースの限られた環境への移植性が高い設計です。動作環境コンパイラ: GCC, Clang, MSVC 等、標準的なC11以降をサポートする環境。数学ライブラリ: コンパイル時に -lm オプションが必要な場合があります。ビルド・実行例Bash# 例: 円の衝突判定プログラムのコンパイル
gcc circle_collision.c -o circle_collision -lm
./circle_collision
今後の課題と拡張浮動小数点の精度誤差への対策: A == (A1 + A2 + A3) の比較において、微小値 epsilon を用いた許容範囲内での一致判定への改良。SIMD化: 大量のオブジェクトを判定する場合のベクトル演算最適化。分離軸定理 (SAT): 回転した矩形や多角形に対応するためのアルゴリズム実装。ライセンスMIT License補足三角形の面積比較による判定 (A == (A1 + A2 + A3)) は、数学的には非常に美しいですが、プログラミングにおいては浮動小数点の計算誤差によって、まれに「内部にあるのに false と判定される」場合があります。もしより「堅牢な」システムを構築されるのであれば、前述のコード（Python版）で用いた 「外積による符号判定（各辺のどちら側に点があるか）」 をC言語に移植すると、より数値的に安定した（かつ除算を必要としない）実装になります。
